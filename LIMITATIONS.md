# REBOUND — Known Limitations

Empirical record. Only measured values from the actual system are
documented here, not estimates or design intentions.

## CNN Classifier (6 classes)

- **val_accuracy**: 0.953 on validation split (20% of dataset, stratified).
  Metrics on the full dataset include training data and are not
  representative.

- **corner/nearby_wall bias**: on the full dataset, the model commits
  26 corner→nearby_wall errors vs. 5 in the reverse direction (~5:1
  ratio). The asymmetry is due to acoustic similarity between both
  classes: strong reflections and low RT60 in both cases.

- **CRITICAL — Simulation-to-reality domain gap**: ALL reported accuracy
  metrics were measured on synthetic data generated with pyroomacoustics
  ShoeBox (image source method). This simulator assumes rectangular
  rooms, flat walls with uniform absorption, and does not model
  diffraction, furniture, or irregular geometry. The "doorway" class is
  simulated as a narrow room, not as a real opening between two rooms.
  The "corner" class is physically similar to "nearby_wall" in ShoeBox.
  Real-world performance on actual RIRs is unknown and likely
  significantly lower. Any accuracy metric should be interpreted as an
  optimistic upper bound, not expected production performance.

- **Stairs class**: trained on synthetic stair RIRs generated by
  `synthesize_stair_rir()` (periodic impulse trains with geometric
  attenuation). Real staircase RIRs include diffuse reflections,
  diffraction at step edges, and wall interactions that the synthetic
  model does not capture. The DSP detector acts as reinforcement —
  CNN + DSP must agree for high-confidence stair detection.

## Stair Detector (two-pass periodic)

- **SNR threshold**: collapse between noise_level=0.018 (SNR 26.9 dB → OK)
  and noise_level=0.019 (SNR 26.4 dB → FAIL). Empirical threshold: ~27 dB.
  Note: noise_level is the absolute standard deviation of Gaussian noise
  in simulate_capture(), not an explicit SNR parameter. Actual SNR
  depends on convolved signal power.

- **Maximum steps**: Pass 2 extension is capped at 40 steps to prevent
  false positives from periodic non-stair patterns (parallel walls,
  acoustic panels).

- **Real deconvolved RIRs**: behavior not evaluated. The system was
  trained and tested exclusively on synthetic RIRs (pyroomacoustics +
  synthesize_stair_rir). Critical risk identified in red team audit.

## API Server

- **Authentication**: no JWT or API key authentication is implemented.
  Endpoints validate user_id format and class_name against a whitelist,
  but do not authenticate callers. Deploying without an authentication
  layer exposes user data.

- **Hardware latency calibration**: `calibrate_hardware_latency()` is
  available but must be called manually before real-time capture. Without
  calibration, distance estimates from `emit_and_capture()` include a
  systematic bias equal to the hardware round-trip latency (typically
  5–50 ms, or 1.7–17 m of range error).

## Feature Extraction

- **Empty mel filters**: with `n_fft=512` at 44100 Hz the resolution is
  86.1 Hz/bin. The lowest mel filters (near fmin=500 Hz) have bandwidth
  smaller than one bin and produce empty responses. Affects the lower
  bands of the mel spectrogram. The bands relevant to the CF-FM chirp
  (8 kHz) are not affected. The corruption is consistent across training
  and inference — the CNN was trained with this behavior.
  Fix: increase n_fft to 2048, regenerate dataset, and retrain. Deferred
  to v2.

- **Noise augmentation**: `add_noise_augmentation` moved to
  `future/augmentation.py`. Bug: noise applied only to mel spectrogram,
  leaving rt60 and spectral_centroid clean — mismatch with real inference.
  Block 1 model was not affected: train.py uses dataset.npz, not
  dataset_augmented.npz. Fix in v2: augment in RIR domain before
  extract_features(). Deferred to v2.

## Training Split

- **Stratified split**: train.py uses a class-stratified split to
  guarantee proportional representation in train and val sets.
  Normalization statistics (rt60, centroid) are computed exclusively on
  the training split to prevent data leakage.

## Deferred Work

- **EWC (Elastic Weight Consolidation)**: implemented in `future/ewc.py`
  but not integrated. Requires DataLoader from the base model and a
  per-user fine-tuning loop. Deferred to v2.
